# 길눈 실제형 의미 안내 데모 설계

## 목표

현재의 단순한 3단계 카드를 실제 공공서비스처럼 정보가 많은 합성 화면으로 바꾸고,
길눈이 좌표가 아닌 `stableKey + role + accessibleName + expectedState`로 현재 해야 할
동작을 다시 찾는 장점을 사용자가 즉시 이해하게 한다.

핵심 문구는 다음으로 고정한다.

> 좌표는 빗나가도, 의미는 다시 찾습니다.

## 선택한 접근

세 기관형 화면을 서로 다른 정보 구조와 시각 언어로 만든다.

- 정부24형: 문서 탭, 안내 상자, 주소 선택, 발급 형태, 수령 방법, 민원 신청 버튼
- 복지로형: 서비스 경로, 신청자 요약, 신청 관계, 확인 항목, 임시 저장과 다음 단계
- 건강보험형: 민원 메뉴, 조회 기준, 자격 안내, 연도 선택, 조회와 일정 안내

실제 기관 로고·로그인·개인정보·제출은 복제하지 않는다. 모든 데이터는 `(가상)` 또는
`(모의)`로 표시하고 실제 기관과 연결되지 않는다는 고정 문구를 유지한다.

## 의미 기반 안내 시연

- 화면에는 관련 없는 메뉴, 탭, 보조 버튼을 함께 배치해 실제 사이트와 비슷한 복잡도를 만든다.
- 각 체크포인트의 진행 대상은 기존 단일 PatchV1 계약을 그대로 유지한다.
- Layout A와 B는 시각적 위치와 그룹 배치를 크게 바꾸지만 DOM 의미 순서와 접근성 이름은 유지한다.
- 자동 안내는 클릭하지 않고 정확히 한 대상만 노란 테두리로 강조한다.
- 홈과 연습 화면에 “버튼 위치가 바뀌어도 이름·역할·다음 상태를 함께 확인한다”는 설명을 노출한다.

## 네이티브 경험

- 앱 시작 시 제공된 길눈 로고가 페이드·스케일로 들어오는 짧은 스플래시를 표시한다.
- 서비스 진입 시 로고와 진행 표시를 포함한 짧은 로딩 상태를 거친다.
- 시스템 애니메이션 배율이 꺼진 경우 즉시 전환하고, 애니메이션은 입력을 막지 않는다.
- 홈은 핵심 가치 제안, “복잡한 화면도 한 단계씩” 설명, 기관형 연습 카드 3개 순서로 구성한다.

## 안전·접근성

- 20sp/20px 이상 본문, 22sp 버튼, 56dp/px 이상 조작 영역을 유지한다.
- 큰 글자에서는 단일 열과 세로 스크롤을 사용하며 가로 스크롤을 만들지 않는다.
- CSP, 오프라인 appassets, URL allowlist, 브리지 strict V2, 정확 일치 및 자동 클릭 금지를 유지한다.
- 실제 기관 API, 실제 로그인, 입력, 제출, 결제, 의료 결과는 추가하지 않는다.

## 검증

- 기존 72개 JVM 테스트를 유지하고 실제형 화면의 복잡도, 핵심 문구, 기관별 구분,
  Layout A/B 위치 변화와 의미 계약 보존을 정적으로 검증한다.
- 스플래시·로딩 상태의 상태 전환 테스트를 추가한다.
- 릴리스 lint, debug/release/androidTest APK, 패키지 자산·CSP·서명·zipalign 전체 게이트를 다시 실행한다.
- 기기가 없으면 모션·TalkBack·200% 글자·실제 WebView 계측은 `DEVICE_BLOCKED`로 남긴다.
